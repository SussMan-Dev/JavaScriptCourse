# JavaScript Course

This repository is a beginner-friendly JavaScript course. Each folder is one lesson, usually containing an HTML file and a JavaScript file that you can open in the browser to practice directly.

The lessons are arranged from basic syntax to browser interaction. Follow the route in order if you are new to JavaScript.

## Study Route

1. `Declaration`
   - Introduction: Learn how variables are created in JavaScript.
   - Main topics: `var`, `let`, `const`, scope, hoisting, and Temporal Dead Zone.
   - Goal: Understand why modern JavaScript usually prefers `let` and `const` over `var`.

2. `DataType`
   - Introduction: Learn the basic kinds of values JavaScript can work with.
   - Main topics: string, number, boolean, null, undefined, object, and basic type checking.
   - Goal: Know what kind of data you are storing and how JavaScript treats it.

3. `console`
   - Introduction: Learn how to print values and inspect code while developing.
   - Main topics: `console.log()`, debugging output, reading values, and checking program flow.
   - Goal: Use the console as your first debugging tool.

4. `condition`
   - Introduction: Learn how to make decisions in code.
   - Main topics: `if`, `else if`, `else`, comparison operators, and logical conditions.
   - Goal: Run different blocks of code depending on different situations.

5. `SwitchCase`
   - Introduction: Learn another way to handle multiple possible cases.
   - Main topics: `switch`, `case`, `break`, and `default`.
   - Goal: Write cleaner condition logic when comparing one value against many options.

6. `function`
   - Introduction: Learn how to group reusable code into named actions.
   - Main topics: function declaration, function call, parameters, arguments, and return values.
   - Goal: Avoid repeating code and start writing more organized programs.

7. `For-WhileLoop`
   - Introduction: Learn how to repeat code automatically.
   - Main topics: `for`, `while`, loop counters, loop conditions, and stopping loops.
   - Goal: Repeat actions without writing the same code many times.

8. `Array`
   - Introduction: Learn how to store multiple values in one variable.
   - Main topics: array syntax, indexes, length, reading items, updating items, and basic array operations.
   - Goal: Work with lists of data.

9. `Object`
   - Introduction: Learn how to store related data using key-value pairs.
   - Main topics: object syntax, properties, dot notation, bracket notation, adding, updating, and deleting properties.
   - Goal: Represent real-world data such as a user, product, or student.

10. `For in + of`
    - Introduction: Learn special loop styles for objects and iterable values.
    - Main topics: `for...in`, `for...of`, object keys, array values, and loop differences.
    - Goal: Choose the right loop when working with objects or arrays.

11. `ForEach`
    - Introduction: Learn a common array method for running code on every item.
    - Main topics: `forEach()`, callback functions, current item, index, and array iteration.
    - Goal: Loop through arrays in a cleaner and more modern style.

12. `map`
    - Introduction: Learn how to transform one array into another array.
    - Main topics: `map()`, callback return values, data transformation, and creating new arrays.
    - Goal: Convert data from one shape into another without changing the original array.

13. `filter`
    - Introduction: Learn how to keep only the array items that match a condition.
    - Main topics: `filter()`, callback conditions, boolean results, and filtered arrays.
    - Goal: Select useful data from a larger list.

14. `DOM`
    - Introduction: Learn how JavaScript connects to HTML.
    - Main topics: `document`, `getElementById()`, selecting elements, and changing page content.
    - Goal: Use JavaScript to update what appears on a web page.

15. `Event`
    - Introduction: Learn how JavaScript responds to user actions.
    - Main topics: `addEventListener()`, click events, form submit events, `preventDefault()`, and input values.
    - Goal: Make web pages interactive.

16. `changingColorButton`
    - Introduction: Practice DOM and events in a small interactive project.
    - Main topics: selecting a button, listening for clicks, checking style values, and changing text color.
    - Goal: Build confidence by connecting user action with visible page changes.

## Practice Projects

After finishing the study route, try building these small projects:

1. A counter with increase, decrease, and reset buttons.
2. A greeting form that displays the user's name on the page.
3. A student list stored as an array of objects.
4. A rendered list using `map()`.
5. A filtered list using `filter()`.
6. A button that changes text, color, or background style.

## JavaScript Overview

JavaScript is a high-level, interpreted programming language used to build interactive web applications. It was created by Brendan Eich in 1995 at Netscape and later standardized by ECMA International under the name ECMAScript.

Key milestones:

- ES3 (1999): foundational version.
- ES5 (2009): strict mode, JSON support, and array improvements.
- ES6 / ES2015: `let`, `const`, arrow functions, classes, and modules.
- ES6+ updates: `async/await`, optional chaining `?.`, nullish coalescing `??`, and many modern improvements.

Strengths:

- Runs in all modern browsers.
- Easy to start learning.
- Strong support for asynchronous programming.
- Huge ecosystem with npm, frameworks, and community support.
- Can be used for both frontend and backend through Node.js.

Weaknesses:

- Dynamic typing can cause unexpected bugs.
- Hoisting and type coercion can be confusing.
- Single-threaded execution requires careful async handling.
- Large projects need good structure to stay maintainable.
- Some legacy behaviors like `var` and `==` should be used carefully.

## How To Study

1. Open each lesson folder in order.
2. Read the JavaScript comments first.
3. Open the HTML file in the browser.
4. Check the browser console when the lesson uses `console.log()`.
5. Change the code and refresh the browser to see what happens.

Learning JavaScript is easier when you experiment. Do not only read the code. Break it, fix it, and try small changes until the behavior makes sense.
